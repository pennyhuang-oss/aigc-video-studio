# coherence — 場景連貫性驗證目錄

此目錄用於 AIGC 影片製作完成後的品質驗證。在進入 G2 素材審核（提交導演審核）之前，**必須執行連貫性驗證**，確保各維度品質分數達到發布標準。

---

## 用途

AIGC 生成的影片素材在以下維度容易出現問題：
- 同角色跨鏡頭外觀不一致（服裝、髮色、膚色）
- 色溫跨鏡頭偏差（暖冷不統一，場景「跳出感」）
- 主體形變（手指、臉部、邊緣輪廓扭曲）
- 音視頻不同步
- 字幕安全區違規（文字壓到主體）

`coherence/` 目錄的驗證腳本自動掃描所有素材，輸出量化分數報告，讓製作人員快速定位問題鏡頭，避免問題素材流入後製剪輯。

---

## 七個驗證維度（D1-D7）

| 維度代碼 | 名稱 | 評分標準 | 滿分 | 最低通過分 |
|---------|------|---------|------|-----------|
| D1 | 角色一致性 | 同一角色跨鏡頭臉部相似度（SSIM）| 10 | 7 |
| D2 | 色溫一致性 | 同場景類型跨鏡頭色溫偏差（K）| 10 | 7 |
| D3 | 主體形變 | 關鍵點輪廓偏移（px）| 10 | 8 |
| D4 | 構圖合規 | 主體位置、字幕安全區符合鐵則 | 10 | 8 |
| D5 | 音視頻同步 | 旁白波形與嘴型對齊誤差（ms）| 10 | 8 |
| D6 | 音量標準 | 最終輸出響度（LUFS）| 10 | 8 |
| D7 | 水印檢測 | 是否存在 Midjourney / Kling 浮水印 | 10 | 10 |

**總分：** 70 分滿分
**G2 通過門檻：** ≥ 56 分（80%），且 D3 / D4 / D7 不低於最低通過分

## 驗證腳本說明

### 目錄結構

```
coherence/
├── README.md                          # 本文件
├── run_coherence_check.py             # 主執行腳本
├── checkers/
│   ├── d1_character_consistency.py   # D1 角色一致性
│   ├── d2_color_temp.py              # D2 色溫一致性
│   ├── d3_motion_distortion.py       # D3 主體形變
│   ├── d4_composition.py             # D4 構圖合規
│   ├── d5_av_sync.py                 # D5 音視頻同步
│   ├── d6_loudness.py                # D6 音量標準
│   └── d7_watermark.py              # D7 水印檢測
└── reports/
    └── [集數]_coherence_report.md    # 自動生成的報告
```

### 執行方式

```bash
# 驗證本集全部素材
python3 run_coherence_check.py --episode EP01

# 只驗證特定維度
python3 run_coherence_check.py --episode EP01 --dimension D3

# 只驗證特定鏡頭
python3 run_coherence_check.py --episode EP01 --shot S03

# 快速掃描（跳過 D5 音視頻同步，速度較慢）
python3 run_coherence_check.py --episode EP01 --quick
```

## 各維度技術說明

### D1 — 角色一致性
使用 SSIM（結構相似性指數）比較同角色跨鏡頭的臉部區域相似度。
- 比較基準：`mj_seed` 鎖定的基準圖（`S03_mj_base.png`）
- 計算範圍：臉部邊界框（face bounding box）內
- 工具：`face_recognition` + `scikit-image`

### D2 — 色溫一致性
提取每個鏡頭的平均色溫（K），與同場景類型的目標色溫比較。
- S-A 目標：5000K（±200K 通過）
- S-B 目標：3800K（±200K 通過）
- S-C 目標：4500K（±200K 通過）
- S-D 目標：5500K（±200K 通過）

### D3 — 主體形變
分析影片中主體關鍵點（人臉 / 手部）的幀間移動量，超過閾值判定為形變。
- 使用 MediaPipe Pose / Face Mesh 偵測關鍵點
- 判定條件：連續 3 幀間同一關鍵點位移 > 10px

### D4 — 構圖合規
自動偵測：主體（人臉）位置是否在上方 1/3 區域，字幕安全區（下方 20%）是否有多餘視覺元素。

### D5 — 音視頻同步
比較旁白音頻波形（語音起始點）與影片中嘴型動作的時間差。
- 工具：`librosa`（音頻分析）+ MediaPipe（嘴型偵測）
- 門檻：誤差 < 80ms 為通過（人眼感知門檻）

### D6 — 音量標準
使用 `pyloudnorm` 計算整體響度，確認符合 -14 LUFS 標準。
- 允許偏差：-13 至 -15 LUFS

### D7 — 水印檢測
使用模板匹配（Template Matching）偵測已知 AI 工具浮水印圖案（Midjourney、Kling、Suno 等）。
- 偵測精確度：99%（基於浮水印固定位置特性）
- 發現水印時：報告精確時間碼 + 截圖
