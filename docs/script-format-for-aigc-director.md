# AIGC 製作師腳本格式規範

**用途：** 提案通過、進入製作階段後，將腳本深度豐富化時使用此格式  
**受眾：** AIGC 製作師（非客戶）  
**觸發時機：** 客戶確認提案方向 → 文字腳本鎖定 → 交給製作師進行分鏡製作前

> ⚠️ 此格式**不用於**客戶提案書。提案書用更簡潔的版本（見 `01-scripts.md`）。  
> 此格式是製作師內部工作文件，細節到可以直接按圖索驥生成素材。

## 格式要素（必須包含）

### 1. 集數 / 版本標題
```
第 X 集｜[集數名稱]
情感階段：[情感標籤，例：觀察 / 改觀 / 心動]
```

### 2. 場景標題
每個場景必須有標題，說明地點與核心動作：
```
場景 1：[地點] + [核心動作簡述]
```

### 3. 時間戳記（每個鏡頭都要有）
```
【0-3s】畫面描述
【3-6s】畫面描述
```
- 每個鏡頭不超過 15 秒（1 cut = 1 鏡頭）
- 時間戳讓製作師知道節奏

### 4. 對白與 OS 分開標示

| 標示方式 | 說明 |
|---------|------|
| `角色名（語氣說明）：「對白」` | 畫面上有人說話 |
| `角色名 OS：「旁白內容」` | 畫外音，不對嘴 |
| `旁白：「旁白內容」` | 第三方敘事旁白 |

### 5. 導演備註標籤（穿插在鏡頭描述內）

| 標籤格式 | 用途 |
|---------|------|
| `【⏰ 開場 X 秒爆擊】` | 節奏控制提醒 |
| `【📌 招牌金句 / 古今對位】` | 剪輯精華提醒 |
| `【💥 視覺巧思核心】` | 特別需要注意的構圖/剪輯設計 |
| `【🔑 伏筆 / 翻牌關鍵】` | 敘事伏筆，演員/製作師必須知道 |
| `【公子今天很美式反差笑點】` | 風格提醒（可自定義品牌標籤）|

### 6. Cliffhanger 標示
每集或每段結尾需標示：
```
🔚 Cliffhanger
角色：「台詞」
```

### 7. 三人組落筆前備註（交付製作師前必填）
文件末尾附上各職能的特別注意事項：

```
✍️ 編劇／腳本
- 本版變動範圍說明
- 特定台詞不可修改的原因

🎬 導演 / 視覺
- 鏡頭構圖特殊要求
- AI Prompt 重點提示
- 必須避免的視覺問題

📊 PM
- 總時長估算
- 核心場景數量
- 優先剪輯的精華片段

⚠️ 製片總控 / 技術
- AI 生成注意事項（例：避免中文手寫字特寫）
- img2img workflow 規則
- 版本變動紀錄
```

### 8. 金句總表（可選，建議加入）
方便剪 6 秒精華 Reels / TikTok 推廣素材：

| 場景 | 金句 | 角色 | 用途 |
|------|------|------|------|
| Ep1 開場 | 「⋯⋯」 | 女主 OS | 開場爆擊 |

## 完整格式範例（以鮨佐樂版本 A 為參考）

```
版本 A｜職人氛圍廣告（16:9 橫式）
情感階段：信任建立 → 渴望 → 行動

場景 1：漁港清晨 — 食材溯源

【0-4s】日本漁港廣角，薄霧，漁船靠岸。藍灰冷調（3800K）。
【⏰ 開場 4 秒建立世界觀，無旁白，音樂帶入】

【4-8s】師傅手部特寫，從保麗龍冰箱取出食材，鏡頭跟著手移動。
【📌 不露臉——職人只有手，留懸念】
旁白 OS：「最好的壽司，從選擇食材的那一刻就開始了。」

場景 2：吧台準備

【8-14s】吧台俯拍，木紋，師傅將食材整齊排列。4500K 中性。
【🔑 節奏放慢，讓觀眾感受「準備」的儀式感】
（無旁白）

🔚 段落轉折
旁白 OS：「日本職人的堅持，不是為了表演——是因為他相信，每一個細節都值得。」

---

✍️ 腳本備註
- 旁白語氣：沉穩、肯定，不煽情，不賣弄
- 「職人」二字不可口語化（避免說成「師傅很厲害」）

🎬 視覺備註
- C01 漁港：AI prompt 重點「薄霧、藍灰冷調、無人臉、廣角」
- 師傅手部：不渲染臉，prompt 加 no face visible

📊 PM 備註
- 版本 A 總時長目標：45 秒
- 精華片段：C03 切魚慢動作 + C08 王爺學貓叫（誤，此為測試用）

⚠️ 製片備註
- 旁白段音量：背景音樂降 6db，確保人聲清晰
- 所有場景 16:9 橫式構圖，主體在畫面中央或上 1/3
```

## 製作流程中的位置

```
客戶提案書（.docx）
    ↓ 客戶確認方向
文字腳本確認版（此格式，內部文件）
    ↓ 製作師確認無誤
AIGC 素材生成（Kling AI / Sea Dance 2 / AI 圖像）
    ↓
剪輯合成
    ↓
交付客戶審閱
```

## AI 生成注意事項（通用規則）

來源：《王妃變貓》製片總控 v14 備註，適用所有 AIGC 影片專案：

1. **禁止特寫中文手寫文字**：AI 渲染中文手寫字崩潰率約 90%，改用 OS 口述 + 視覺暗示代替
2. **img2img prompt 規則**：用最短句描述（「半身近照」「3/4 側身」），不重述人物特徵
3. **角色一致性**：建立 5 張固定 reference（不同姿態），每集生成前先對照
4. **情緒觸發邏輯**：台詞/OS 描述情緒變化時，要在 prompt 裡對應反映（非直接翻譯台詞）
