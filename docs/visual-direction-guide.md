# 視覺導演規格 — 豎屏三鐵則 + 色溫方案

---

## 豎屏三鐵則（不可妥協）

### 鐵則 1：主角永遠在畫面上 1/3

人眼在豎屏上的視線習慣是由上往下掃，而非置中凝視。

- **規格：** 臉部眼睛落在畫面頂端 30% 的區域內
- **1080×1920 基準：** Y 軸 576px 以上
- **違規例外申請：** 一律退回，不以「美觀」或「構圖完整」為由例外

### 鐵則 2：字幕在安全區內，不壓臉

- **字幕區域：** 畫面下方 20%（約 384px 高度空間）
- **字體大小：** ≥52pt（1080p 輸出標準）
- **行距：** 1.4 倍
- **禁止：** 字幕與主角臉部重疊（尤其不能蓋住眼睛）
- **B-roll 有截圖時：** 字幕加半透明底板，可讀性 ≥ 90%

### 鐵則 3：CTA 最後 10 秒全屏

- **背景：** 品牌主色（全屏）
- **CTA 文字：** 置中，≥80pt
- **語氣：** 乾淨、自信地複述一遍
- **只允許：** 一個動作 + 一個連結
- **禁止：** 多條資訊、動態特效干擾

## 色溫方案（三段）

| 段落 | 色溫 | 情緒 | 適用場景 |
|------|------|------|---------|
| 開場痛點 | 3800K（藍灰冷調）| 緊張、壓迫、孤獨 | 深夜場景、帳單特寫、失控感 |
| 採訪主段 | 5000K（中性）| 真實、可信、自然 | 人物採訪、轉折發現 |
| 結果收尾 | 5500K（暖白）| 解脫、滿足、自信 | 成果展示、情感高光、CTA |

**驗收方式：** 三段截圖並排，應能目視感受從冷到暖的遞升。並排看不出差異 = 調整不足。

## 攝影機運動規範（豎屏）

**優先使用：**
- 垂直 push-in
- 靜止穩定
- 輕微 zoom-in（上方主體）

**禁用：**
- 橫向 pan（豎屏比例視覺不適）

**Kling AI 豎屏動態 Prompt 補充語：**
```
vertical 9:16 composition, slow vertical push-in, subject in upper frame,
lower area kept clean for text, mobile-first framing
```

## CTA 最佳實踐

**靜態 URL 大字 vs 動態打字：**
- 優先使用：搜尋列打字動畫（行動暗示更強）
- 備選：靜態 URL 大字

**片尾視覺建議：**
- 吉祥物 / 品牌 Logo 主視覺
- 網域文字
- 副文案（免費試用 / 核心賣點 1~2 條）
