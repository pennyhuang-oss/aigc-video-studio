# 製作框架總覽 — 5 Phases + 4 QC Gates

---

## 整體流程

```
[新專案啟動]
      ↓
  Phase 0：啟動 & 四鎖確認
      ↓
  G0 審核：立案通過（含預算 + 規格確認）
      ↓
  Phase 1：腳本撰寫
      ↓
  G1 審核：腳本品質確認（品質檢核 + 創意總監）
      ↓
  Phase 2：AIGC 素材生成
      ↓
  G2 審核：素材技術規格 + 版權確認
      ↓
  Phase 3：後期剪輯（字幕 + 特效 + 合成）
      ↓
  G3 審核：精剪版本確認（創意總監 + 品質檢核）
      ↓
  Phase 4：多平台格式輸出 + 最終 QC
      ↓
  G4 最終審核：品質得分 ≥ 90 才允許發布
      ↓
  [各平台發布]
```

## Phase 0 — 專案啟動

**目標：** 完成「四鎖」，產出 G0 製作 Brief

### 四鎖（開工前必須全部鎖死）

#### 鎖 1：成片規格
在 G0 由「真正下指令生產的人」確認，寫進 Brief 第一行。

| 規格項 | 選項 | 確認值 |
|--------|------|--------|
| 畫面比例 | 9:16（豎式）/ 16:9（橫式）/ 兩者 | |
| 目標時長 | 30s / 60s / 90s / 120s / 其他 | |
| 幀率 | 24fps / 30fps | |
| 主推平台 | TikTok / IG / YouTube Shorts / LinkedIn | |
| 解析度 | 1080×1920 / 1920×1080 | |

> ⚠️ 規格不確定就先做規格決策，不要先寫鏡頭。一份為「2分鐘直式」設計的分鏡，遇到「78秒橫式」，構圖與節奏設計幾乎全廢。

#### 鎖 2：產品真相表
開工前必須有，內容來自真實產品（不是想像）：

| 欄位 | 說明 |
|------|------|
| 這支片賣什麼（一句話）| 核心價值主張，必須對齊真實產品定位 |
| 真實 UI 截圖 | 由產品/工程師提供，非設計稿概念圖 |
| 真實定價邏輯 | 用量制 / 訂閱制 / 混合，核心數字 |
| 真實證明素材 | 截圖、email、報表，非 GFX 抽象文字 |

#### 鎖 3：品牌資產表
向品牌方取齊：

- [ ] 吉祥物（有/無、用法規範）
- [ ] Logo 各版本（全彩/白/黑/深底）
- [ ] 浮水印規範
- [ ] 色票（精確色碼，容差要求）
- [ ] 字體（中英文各一）
- [ ] 網域、email 格式、slogan

#### 鎖 4：製作方式
- [ ] 全 AIGC
- [ ] 真人拍攝
- [ ] 混合（指定哪些鏡頭真人，哪些 AIGC）

**決定後立刻刪除用不到的協定。** 確認全 AIGC 後，刪除所有真人採訪協定。

## Phase 1 — 腳本撰寫

**G1 提交格式（缺少任何一項則退回）：**

- [ ] 情緒溫度標記（每場景 0~100 值）
- [ ] 具體可量化的改變（金額 / 時間 / 人數）
- [ ] 主角個人動機（財務壓力 / 職業責任 / 對失控的恐懼，「公司要求」不算動機）
- [ ] CTA 銜接邏輯（最後 10 秒如何從故事自然引出）

**三段式敘事結構：**

| 段落 | 時長目標 | 情緒溫度目標 | 任務 |
|------|---------|------------|------|
| 痛點鋪陳 | ~30~45 秒 | 20 → 65 | 建立可感受的場景，非一句台詞 |
| 見證故事核心 | ~40~80 秒 | 65 → 90 | 至少一個具體可量化的改變 |
| 產品展示 + CTA | ~20~40 秒 | 90 → 70 | 展示對應見證說的那個轉變；最後 10 秒只有一個動作 |

## Phase 3 — 後期剪輯

**工序清單：**
- [ ] Kling AI 動態化批次生成
- [ ] HeyGen 口型同步（SOT 鏡頭）
- [ ] 色溫調色（三段大方向：冷 / 中性 / 暖）
- [ ] After Effects GFX 疊加
- [ ] Suno AI 音樂四段剪接（痛點 / 轉折 / 使用 / 結果）
- [ ] 字幕製作（Noto Sans TC Bold，安全區內，不壓臉）

## Phase 4 — 多平台格式輸出

| 平台 | 格式 | 解析度 | 幀率 | 碼率 | 音頻 |
|------|------|--------|------|------|------|
| TikTok | MP4 H.264 | 1080×1920 | 30fps | ≥8 Mbps | AAC 44.1kHz -14 LUFS |
| Instagram Reels | MP4 H.264 | 1080×1920 | 30fps | ≥8 Mbps | AAC 44.1kHz -14 LUFS |
| YouTube Shorts | MP4 H.264 | 1080×1920 | 30fps | ≥8 Mbps | AAC 44.1kHz -14 LUFS |
| LinkedIn | MP4 H.264 | 1920×1080 | 30fps | ≥8 Mbps | AAC 44.1kHz -14 LUFS |

> YouTube Shorts 上限 60 秒，若主版本超過 60 秒，需另剪精華版。

## 時程規劃參考（以 5 集見證影片為基準）

| Phase | 工作天（估算）| 主要阻塞點 |
|-------|------------|-----------|
| Phase 0 + G0 | 3~5 天 | 四鎖會議、預算核決 |
| Phase 1 + G1 | 7~14 天 | 腳本定稿（含大綱確認 + 全稿） |
| Phase 2 + G2 | 14~21 天 | CHAR 角色 Seed 鎖定、UI 截圖交付 |
| Phase 3 + G3 | 10~15 天 | HeyGen 口型同步品質 |
| Phase 4 + G4 | 5~7 天 | 多平台格式輸出 |
| **總計** | **39~62 天** | |

## Gate 審核標準

| Gate | 審核者 | 通過條件 |
|------|--------|---------|
| G0 | 創意總監 + 製作統籌 | 四鎖確認完畢、預算核決 |
| G1 | 品質檢核 + 創意總監 | 腳本格式完整、情緒弧線合理 |
| G2 | 品質檢核 | 素材技術規格合規、版權確認 |
| G3 | 創意總監 + 品質檢核 | 精剪版本完整、品牌一致性 |
| G4 | 最終核決者（通常為品牌方）| 品質得分 ≥ 90 |
