# 見證影片製作通用 Skill

**適用：** 所有 B2B / B2C 品牌見證影片、案例研究影片  
**來源：** AITokenKing EP.01 成片 vs 規劃落差分析（19-ep01-plan-vs-result-gap-analysis.md）  
**版本：** v1.0（2026-06-25）

> **這份 Skill 是怎麼來的？** EP.01 成片拿了 90 分，但它和製作團隊的規劃只有約六成吻合。那 40% 的落差（不相符 / 沒思考到 / 捨去 / 多講）就是這份 Skill 的全部內容。**不是理論，是已經做錯 / 差點做錯的東西。**

## 二、敘事與腳本

- **定位一句話先行。** 寫腳本前先確認「這支片到底賣什麼」
- **痛點對齊真實買家的真實痛。** B2B 見證的痛點是「治理盲區 / 報帳麻煩 / 不知值不值」，不是戲劇化的深夜崩潰
- **招牌台詞不保證存活。** 金句是加分，敘事骨架才是命。不要把全片成敗賭在一句金句
- **數字用真實可證的。** 「便宜 41%」比「從 NT$27,000 降到 NT$16,000」更保險（後者需要真實憑證）

---

## 三、鏡頭與節奏

- **SOT 連續段上限：~12 秒 / 最多 2 個 AI Avatar 片段**（超過就拆景別或插 B-roll）
- **B-roll 上的 SOT：≤1 句**（超過就要切回人臉）
- **UI 展示段之間插人臉反應鏡頭**（2~3 秒），避免純 UI 段落流失注意力
- **開場先建立情境、再引入人物**（第一個鏡頭不切人臉效果好）
- **情感降落段：景別只能維持或拉遠，不可從 CU 切回 MCU**
- **明快優先。** B2B 社群見證，78 秒的完播率可能優於 2 分鐘。長版另出，不要把長版當預設

---

## 四、視覺與證明

- **證明型內容用真實介面截圖，不用抽象 GFX 文字**（發票 email 截圖 > 「月結統一發票」標籤）
- **UI 用真實樣貌，不要美化成想像的 mockup**（真實數據表 > 想像的漂亮介面）
- **用「被介紹 / 被簡報」場景帶出產品全貌**（比「手機搜尋」更自然且製作感高）
- **色溫守「冷開場、暖收尾」即可**，不做逐鏡精密 LUT
- **全片品牌浮水印**（低成本高記憶點），片尾用吉祥物主視覺
- **CTA 用動態（搜尋列打字 URL）取代靜態大字**，行動暗示更強

---

## 五、規劃顆粒度收斂（不要做的過度設計）

| 不要做的 | 改成 |
|---------|------|
| 14 點精密情緒溫度 + 每步 ≤20 檢查 | 只守「冷開場→暖收尾」三錨點 |
| 逐鏡 IP-Adapter 強度 + Fixed Seed 六步流程 | 角色一致性用「同參考圖 + 抽查」 |
| J-cut / L-cut 14 條轉場編排表 | 預設硬切，特殊處才標 |
| 五階色溫 LUT 逐鏡 | 三段大方向 |
| 真人採訪協定（全 AIGC 時）| 製作方式決定後刪除 |
| CTA 前 0.7s 靜幀 hold 專章 | 視成片需要再加 |

**判準：這條規格，最後真的會有人照著做、且做得到嗎？**

## 六、品牌一致性（G4 必查）

- 所有畫面內嵌 URL = 真實網域（含 .com.tw 等後綴）
- 所有年份對齊上線年度
- 色碼零容差（背景色、強調色）

## 七、四類落差速記（每支成片完成後用這四欄回顧）

| 類型 | 定義 | 對 Skill 的意義 |
|------|------|----------------|
| 【不相符】| 規劃了但做出來不一樣 | 校準規劃假設 |
| 【捨去】| 規劃了但成片沒有 | 當初就是多規劃的 |
| 【沒思考到】| 成片有但規劃沒提 | 開工前要先盤點的真實資產 |
| 【多講】| 規劃過度落地用不上 | 收斂顆粒度 |
