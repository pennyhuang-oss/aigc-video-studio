# 影音製作企劃室 — 工作憲章（Crew Charter）

**版本：** 1.0
**適用範圍：** 所有品牌宣傳片、社群影音內容製作專案
**最後更新：** YYYY-MM-DD

---

## 一、核心理念

**我們製作的不是廣告，是打動人心的真實故事。**

每一支影片都是一次溝通。觀眾在 0.5 秒內決定要不要繼續看，在 5 秒內決定是否認同，在 30 秒內決定是否採取行動。我們的工作是讓這三個決策點都答「是」。

**企劃室工作原則：**
- 問題優先，方案其次：先確認我們在解決什麼問題，再討論怎麼做
- 數據驅動，直覺輔助：用數據確認假設，用直覺發現機會
- 快速迭代，不追求完美：能拍出去的 80 分，優於沒拍出去的 100 分
- 品牌一致性高於個人風格：個人創意在品牌框架內發揮

## 二、核心角色定義

### 導演（Director）
**職責：** 整體視覺語言、敘事節奏、最終影片品質把關

**決策範圍：**
- 場景視覺風格最終定案
- 鏡頭語言與剪輯節奏
- AIGC 素材的取捨與重做決定
- 上線前最終審核

**導演不負責的事：**
- 排程與時程管理（PM 負責）
- 文案與腳本初稿（編劇/創意負責）
- 平台技術規格（製片總控負責）

### 編劇 / 創意（Writer / Creative）
**職責：** 腳本、文案、核心創意概念

**決策範圍：**
- 腳本結構與對話
- 開場鉤子設計
- 各平台版本文案
- 角色設定與口吻

**輸出物：**
- `02-script-vX.md`：腳本
- `03-visual-script-EPxx.md`：分鏡腳本
- 各版本發布文案（填入 `05-publish-tracker.md`）

### 製作統籌（PM / Production Manager）
**職責：** 時程、資源調度、跨角色協調

**決策範圍：**
- 製作時程與 Milestone 設定
- 外部資源（配音、音樂授權）
- 問題上報機制

**PM 核心工具：**
- `12-aigc-workflow-EPxx.md`：工作流追蹤
- `05-publish-tracker.md`：發布排程管理
- Kanban / 任務管理工具

### 製片總控（Production Supervisor）
**職責：** 技術規格把關、品牌資產管理、平台合規

**決策範圍：**
- 最終輸出格式與技術規格
- 品牌視覺一致性（Logo 使用、色彩）
- 平台政策合規（版權、廣告標示）
- AIGC 工具選用（MJ vs Seedream 等決策）

## 三、討論機制六項鐵律

### 鐵律一：一次只討論一個版本
每次討論聚焦在「當前版本」的具體問題，不在同一次討論中跳到「如果重新來過」的假設情境。改變方向需要明確宣告並獲得全員確認。

### 鐵律二：反饋必須具體，不接受模糊評語
❌ 無效反饋：「感覺不對」、「可以更好」、「我不喜歡」
✅ 有效反饋：「第 8 秒的鏡頭切換太快，建議延長至 1.5 秒」、「開場文案太長，超過 15 字，建議刪至 12 字以內」

每條反饋必須包含：**[時間點/位置] + [問題描述] + [具體建議]**

### 鐵律三：導演有最終視覺決策權
視覺相關爭議（鏡頭語言、剪輯節奏、色調）由導演拍板。其他角色可以提出理由，但導演決定後不再翻案，除非發現新的客觀問題。

### 鐵律四：數據不支持的「感覺」需要標注
討論中若以個人感覺為依據，需明確說明「這是個人直覺，無數據支持」。下一版本測試後，以數據驗證直覺是否正確。

### 鐵律五：版本號遞增，不覆蓋舊版本
所有文件與素材使用版本號（v1 / v2 / v3），不覆蓋舊版本。每次重大修改提升一個版號。這讓我們可以回滾，也讓審核歷史清晰。

### 鐵律六：截止日前 24 小時不做大改動
距離發布 24 小時內，只接受「必要的技術修正」（字幕錯字、音量問題），不接受「創意方向調整」。創意改動需推遲至下一集。

## 四、製作流程 Gate 系統

每個 Gate 是一個里程碑，必須通過才能進入下一階段：

| Gate | 名稱 | 通過條件 | 負責審核 |
|------|------|---------|---------|
| G0 | 企劃確認 | 品牌 Brief 齊全，目標受眾、KPI 已定 | 製片總控 |
| G1 | 腳本確認 | 腳本通過雙層檢查（見第五節），品牌方已簽核 | 導演 + 品牌方 |
| G2 | 素材確認 | 所有 AIGC 素材生成完畢，通過 coherence 驗證 | 導演 |
| G3 | 初剪確認 | 初版影片完成，音視頻同步，節奏符合腳本 | 導演 + PM |
| G4 | 終剪確認 | 細節修正完畢，技術規格通過，品牌方最終確認 | 製片總控 |
| G5 | 發布準備 | 各平台版本輸出完畢，文案已備，UTM 已設定 | PM |

## 五、雙層腳本 / 分鏡檢查表

### 第一層：創意層檢查（G1 前）

**敘事結構：**
- [ ] 開場 5 秒是否有足夠的鉤子（問句 / 衝突 / 數字衝擊）
- [ ] 目標受眾是否能在前 3 秒認同「這是在說我」
- [ ] 痛點描述是否真實、不誇大
- [ ] 解決方案展示是否清晰（觀眾明白「產品能解決什麼」）
- [ ] 成果描述是否具體（有數字 / 有情境，非模糊形容詞）
- [ ] CTA 是否清晰（觀眾知道「下一步要做什麼」）

**平台適配：**
- [ ] 版本 A（TikTok）：前 3 秒無需靜音也能理解
- [ ] 版本 B（快剪）：30 秒內完整傳達核心訊息
- [ ] 版本 C（LinkedIn）：調性是否適合 B2B 受眾

**品牌一致性：**
- [ ] 品牌色彩是否出現在關鍵畫面
- [ ] 品牌 Logo 出現時機是否符合規範
- [ ] 語氣口吻是否符合品牌聲音語言（07-audio-design.md）

### 第二層：技術層檢查（G4 前）

**視覺技術：**
- [ ] 解析度：1080×1920 px
- [ ] 幀率：24fps
- [ ] 無水印、無 AI 生成痕跡（明顯形變）
- [ ] 字幕位置：下方 20%，不壓主體
- [ ] 各場景色溫符合規格（10-scene-design.md）
- [ ] S-D 場景 Film Grain 已加入（σ=8~12）

**音頻技術：**
- [ ] 整體響度：-14 LUFS
- [ ] 旁白無爆音（峰值 < -1 dBTP）
- [ ] 音樂 Ducking 流暢
- [ ] 品牌 Stinger 出現時機正確
- [ ] 通過 coherence/ 目錄自動化驗證

**合規：**
- [ ] 無侵權音樂（使用 Suno AI 生成或授權素材）
- [ ] 廣告標示符合平台規定（如有付費推廣）
- [ ] 無誤導性聲稱（效果數字可驗證）

## 六、品質底層基本功

### 視覺品質
**什麼是「夠好」的 AIGC 素材：**
- 主體輪廓無明顯形變（容忍：< 10px 偏移）
- 色溫統一（同場景跨鏡頭偏差 < 200K）
- 無水印、無文字疊加
- 構圖符合 9:16 鐵則（主體上 1/3，字幕安全區乾淨）

**什麼是「必須重做」：**
- 手指數量錯誤、面部明顯扭曲
- 色溫偏差 > 500K（跳出感強烈）
- 有 Midjourney / Kling 品牌浮水印
- 主體被畫面邊緣截斷

### 敘事品質
**完播率心法：**
1. **前 3 秒**：視覺衝擊或文字鉤子，讓觀眾停下來
2. **5-15 秒**：建立情感認同（「你說的就是我的情況」）
3. **15-45 秒**：展示解決方案（簡潔、具體、可信）
4. **45-90 秒**：社會認證 + 成果（數字 / 真實反應）
5. **最後 5-10 秒**：清晰 CTA（一個行動，不是三個選項）

**下架風險清單（必須規避）：**
- 誇大療效 / 財務收益（「保證賺錢」、「一定有效」）
- 未標示廣告的商業內容
- 使用未經授權的他人素材（音樂 / 圖片 / 影片）
- 個資展示（他人電話、地址、帳號截圖）

## 七、溝通與文件規範

### 檔案命名規範
```
腳本：[品牌代碼]-EP[XX]-script-v[版本號].md
分鏡：[品牌代碼]-EP[XX]-visual-script-v[版本號].md
素材：[品牌代碼]_EP[XX]_[鏡號]_[工作流]_v[版本號].mp4
最終影片：[品牌代碼]_EP[XX]_[版本A/B/C]_[平台]_v[版本號].mp4
```

### 版本號規則
- v1：第一稿，可能有明顯問題
- v2：修正主要問題後
- v3+：細節調整
- Final：品牌方簽核後，不再修改

### 會議記錄
每次製作會議必須留下文字記錄，包含：
- 會議時間、參與者
- 討論的決策事項（已定案）
- 待確認事項（負責人 + 期限）
- 下次會議時間

## 八、異常處理機制

### 時程延誤
| 延誤程度 | 觸發條件 | 處理方式 |
|---------|---------|---------|
| 輕微（< 1 天）| 單一工作項目延誤 | PM 自行調整，告知導演 |
| 中度（1-3 天）| 影響發布日期 | PM 召集緊急會議，評估是否延後發布 |
| 嚴重（> 3 天）| 需砍功能或版本 | 全員決策，品牌方告知 |

### 品質問題
| 問題類型 | 觸發條件 | 處理方式 |
|---------|---------|---------|
| AIGC 素材品質不足 | 形變 / 色溫嚴重偏差 | 重新生成，不妥協發布 |
| 腳本方向偏差 | G2 後發現敘事問題 | 評估是否值得重拍或調整剪輯 |
| 品牌方大改需求 | G4 後大幅修改 | 新開版本號，評估時程影響 |

### 緊急發布
若有時事熱點需要快速反應：
1. 導演評估是否需要新腳本或借用既有素材
2. PM 確認最短可行時程（最快 48 小時出片）
3. 降低品質要求到：G1 腳本確認 + G3 初剪確認（跳過 G2 coherence 驗證）
4. 發布後補做完整 QC 記錄
